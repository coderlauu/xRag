# v5 Handoff

**日期：** 2026-04-11
**版本：** `v5 / Phase 2B`
**状态：** active

---

## 1. 当前版本一句话目标

在 `Phase 2A` 可信问答闭环完成后，把 `Phase 2B` 的 `Lane 0` contract-to-code 与 `Lane B` worker evidence path 落到代码事实源，并继续推进 `Lane A`，随后进入 `Lane C / D`。

---

## 2. 本轮边界

### 必须实现

- `P0-01` 扩展范围控制：`标签 / 来源类型 / 时间范围`
- `P0-02` 检索实验台基线：`lexical / semantic / rerank` 贡献、未入答原因、URL state
- `P0-03` 证据包增强：claim grouping、多引用组合、freshness 提示强化
- `P0-04` 问题历史与继续提问：最近问题、旧 session 回看、显式继承的 follow-up
- 对应的 `PRD / backlog / interaction delta / technical tradeoff / architecture / data-model / api / contract-freeze` 已形成当前事实源

### 本轮次级目标

- `P1-01` 评估与运维面板增强：`recall / groundedness / citation coverage / latency / cost / backlog`
- 若排序冲突，优先保证主问答入口的范围、证据、检索解释和 history 四条主线先冻结

### 明确不做

- 重新打开 `Phase 2A` 已完成的实现 lane
- 在 `Lane A / B / C / D` 中擅自重定义 `Lane 0` 已冻结的 API、schema 或 shared-types
- 开放互联网联网回答
- 浏览器插件
- 团队协作 / 权限模型
- 移动端
- 全自动 agent 工作流

### 原型 / 文档中存在但本轮只做预留

- 单文档摘要
- 多文档对比摘要
- 相关内容推荐
- 搜索高级过滤器增强
- 多模型路由与更细粒度模型策略

## 6. 输出要求

- 已把 `Phase 2A` 的 `P1` 与 `Phase 2B+` 候选能力整理成新的版本范围
- 已冻结 `must-have / optional / deferred`，避免把后续能力重新挤回首批范围
- 当前最小事实源包括 `PRD / backlog / interaction delta / technical tradeoff / architecture / data-model / api / contract-freeze`，以及已落地的 `schema / shared-types / OpenAPI / SDK`
- 保持 `citation / refusal / freshness / release-readiness` 四条现有信任边界不被新需求破坏
- `Lane 0` 与 `Lane B` 已落地；`worker` 已完成 `typed scope filters / answer_claims / claim-level citations / low_support exclusion`，后续 lane 只能在既有 contract 上实现，不重新打开 source-of-truth

---

## 7. 进入并行实现前必须锁定的问题

1. `Lane A` 是否继续保持不改 API path、字段命名、状态枚举和主数据模型语义？
2. `Lane C / D` 是否只消费 `Lane 0 / Lane A / Lane B` 已稳定的 shape，而不自行猜测后端结构？
3. 测试 lane 是否仍只在 `Lane A / C / D` 合流后启动？
4. `ops / P1-01` 是否继续保持 freeze-late，不反向拖慢 `P0` 主链？
5. 并行实现一旦出现 contract 漂移，是否立即切回主线程收口？

---

## 8. 已知风险 / 历史决策

- `Phase 2A` 已完成可信问答闭环，任何新版本都不能削弱引用、拒答、freshness 与发布可信度
- `Phase 2B+` 候选能力较多，若不先冻结边界，极易出现范围膨胀
- `question history` 很容易演化成多轮 assistant；若没有新的 trust model，不应默认扩张
- `retrieval workbench` 与 `ops metrics` 既有产品价值也有内部诊断价值，必须先定义主用户与验收方式
- `Lane 0` 与 `Lane B` 已落地，但 `apps/web/src/lib/api.ts`、`packages/shared-types/src/index.ts`、`apps/worker/src/database/answer-repository.ts` 仍是热点文件，后续 lane 需要严格遵守 ownership
