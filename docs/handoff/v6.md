## 8. 已知风险 / 历史决策

- GitHub Actions run `24514690725` 仍是当前已验证的 `Phase 2C` implementation lanes main 绿态基线；新版本不得破坏 `citation / refusal / freshness / release-readiness`
- `Lane 0` 集成测试曾暴露 worker constants 对 `@xrag/shared-types` 的运行期 re-export 会导致 `dist-integration` 加载源码 TS；已改为 worker 本地运行期常量定义并保留 shared-types union type 约束，queue/job 字面量未改变
- `Lane C` 已刻意把 notices 做成 prompt-only 提示，不自动阻断 Ask / Search / Detail 当前行为；若 `Lane D` 为了测试便利引入硬拦截，会直接破坏已冻结 contract
- GitHub Actions run `24542756511` 已确认失败；根因在 `smoke-production` job 未安装 workspace 依赖即执行 `node scripts/write-deployment-record.mjs`，导致 `Cannot find module 'pg'`
- 在修复提交及其后续 main CI 绿灯前，不应把 `Phase 2C` 状态写成 `completed`
- 目标环境中仍可能存在历史 `not_indexed` 文档，但这是运行态治理输入，不默认等于 `v6` 的产品范围
- 若 implementation lanes 阶段把自动 remediation、日志平台或多模型治理一起带入，`v6` 很容易重新陷入范围膨胀
