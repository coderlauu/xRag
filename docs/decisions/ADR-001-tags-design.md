# ADR-001: Phase 1 标签系统设计决策

**日期：** 2026-03-31
**状态：** 已决定
**涉及：** 原型 v1-phase1a / 导入页标签交互

---

## 背景

原型评审中发现，原有的 free-text 标签输入框体验不合理：
- 用户容易写错、写重
- 导入时每次都要手动输入，摩擦大
- 没有统一管理入口

---

## 决策

### 1. 标签列表改为预定义选择制

**变更内容：**
- 导入页不再使用 free-text 输入框
- 改为：展示已有标签列表 → 用户点击选择（支持单选/多选）
- 同时保留「输入新标签」入口：用户输入新标签名并回车，自动加入列表并选中

**来源：**
- 初始标签从现有文档标签汇总（见下方）
- 用户可随时新增
- 新增的标签自动进入标签库

### 2. 标签结构维持扁平（Phase 1）

**结论：** 不做层级标签（一级/二级/三级）

**理由：**
- MVP 阶段标签核心目的是「快速筛选」，不是「知识分类」
- 层级标签拉高导入时的决策成本
- 标签数量膨胀到 30+ 前，平铺完全够用

**后期替代方案（已预留）：**
如果后期标签膨胀，用「标签分组」替代层级标签：
- 用户仍然打平铺标签
- 选择时按分组过滤（如 [全部] [用户研究] [技术] [产品] ...）
- 不限制单个文档的标签数量

### 3. 删除标签的行为

**结论：** 软删除，保留孤立标签

**行为：**
- 用户删除标签时，从可用列表移除
- 已导入文档中的该标签记录**保留**（不级联删除）
- 被删除的标签在列表中不再显示（孤立状态）
- 后续如重建同名标签，历史数据可恢复

**理由：**
- 避免误操作导致历史文档标签突然消失
- 数据完整性优先

### 4. 初始标签清单

从原型 mock 数据汇总：

```
用户研究, 访谈, 检索, MVP, RAG, 搜索, 交互, 财务, OCR, 行业研究, 待处理, 链接, P1, 周报, 产品策略
```

（共 15 个，后续随使用持续扩充）
