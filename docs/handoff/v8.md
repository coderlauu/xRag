## 2. 本轮边界

### 必须实现

- 按 `new-version` 启动 `v8 / Phase 3B`，保持 `v7 / Phase 3A` completed baseline 不再扩 scope
- 已完成 planning-and-scope：明确 product scope、P0/P1、交互入口、技术 tradeoff、contract freeze 前置条件和验证策略
- 已完成 technical-evaluation：评估 recovery candidate、operator action、execution audit、rollback plan 的 source-of-truth、API/schema/shared-types/queue 影响和 contract freeze 前置条件
- 已完成 contract-freeze：锁定 schema、shared-types、API path、DTO、OpenAPI/API client 义务、recovery action 状态机、queue 对账和 answer-quality 边界
- 已完成 implementation-freeze：锁定 Lane 0 主线程写入范围、后续 API/Web/Test lane 拆分、验证矩阵和 rollback point
- 当前进入 `testing-and-release-readiness`：`Lane 0: Contract To Code`、`Lane A: API Recovery Candidates, Preview, Rollback Plan`、`Lane B: API Recovery Action Execution And Reconciliation`、`Lane C: Web Ops Recovery Workflow` 与 `Lane D: Integration, E2E, Smoke` 已完成，下一步执行 release-readiness 验证矩阵与 closeout 收口
- 将 `v7` 的 diagnostic samples、answer replay、document replay、deployment compare 作为恢复候选的事实来源，不并行造新的诊断影子数据
- 规划 `P0-01 Recovery Candidates`：从诊断样本生成可解释的候选恢复动作，包含来源、影响对象、风险、前置条件和 preview/dry-run 结果
- 规划 `P0-02 Operator-approved Rerun`：只允许人工确认后的受控 rerun，优先覆盖 document indexing rerun 或 answer diagnostic rerun
- 规划 `P0-03 Recovery Execution Audit`：每次 operator action 必须有 action id、actor、target、reason、before/after facts、status、error 和时间戳
- 规划 `P0-04 Guarded Rollback Planning`：第一版只产出 rollback plan、deployment compare、manual checklist，不做自动 rollback
- 继承 Ask active-session stuck polling 复盘中的异步状态约束：任何新长任务或队列动作都必须有服务端终态收口、queue 对账和前端停止条件
- 如果 v8 需要 schema、shared-types、API path、DTO、OpenAPI、API client 或状态机变更，必须先进入 `contract-guard`
- 如果 v8 触碰 retrieval、answer、citation、freshness、refusal 或 eval 语义，必须先进入 `answer-quality-gate`
- production 历史 `not_indexed` 治理保持为版本外运维动作；如需执行，先单独跑 `pnpm recovery:backfill-indexing -- --dry-run` 并把结论反馈到 scope，而不是直接并入 P0

### 明确不做

- 不继续扩大 `v7 / Phase 3A` scope
- 不绕过 `contract-guard` 修改 Lane 0 已落地的 schema、shared-types、DTO、OpenAPI、API client 或状态枚举
- 不绕过 contract freeze 修改 schema、shared-types、DTO、OpenAPI、API client 或状态枚举
- 不做全自动 remediation、全自动 rerun 或全自动 rollback
- 不把 rollback 执行权直接交给系统；第一版只做人工可审查计划与 checklist
- 不开放互联网联网回答
- 不引入团队协作、权限模型、浏览器插件或移动端
- 不把多模型治理混入 `v8 / Phase 3B` P0
- 不改写 Ask/Search/Detail 主链路响应 contract，除非后续 contract freeze 明确批准

### 原型 / 文档中存在但本轮只做预留

- 多模型 routing、模型成本治理、模型策略切换
- 团队协作权限、审批流和多角色审计
- 自动 remediation agent
- 自动 production rollback
- 更深的知识库管理、推荐、摘要与联网回答
