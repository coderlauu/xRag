## 3. 导入页 / Inbox

### 页面目标

让用户尽可能低摩擦地把资料收进系统。

### 主要模块

1. Hero 区
   - 呈现当前版本目标
   - 强调 `导入 -> 处理 -> 找回`

2. 指标卡
   - 总文档数
   - 可检索数量
   - 处理中数量
   - 失败数量

3. 文本导入表单
   - 标题
   - 正文
   - 标签
   - 保存按钮

4. 文件上传区
   - 选择文件
   - 支持类型说明
   - 演示按钮
   - 重置按钮

5. 最近导入记录
   - 展示最近文档
   - 支持状态筛选
   - 支持跳转详情和搜索

6. 处理队列
   - 展示处理中或失败项

### 核心交互

#### 3.1 文本保存

- 用户填写正文后点击保存
- 系统创建一条文档记录
- 文档状态直接为 `success`
- 页面出现 toast 提示
- 最近导入区刷新

#### 3.2 文件上传

- 支持 `.txt`、`.md`、文本型 `.pdf`
- `.txt / .md` 直接进入 `success`
- `.pdf` 进入 `processing`
- 类扫描件 PDF 进入 `failed`

#### 3.3 最近导入记录筛选

- 支持 `all / success / processing / failed`
- 只影响展示层，不改变数据

### 状态说明

- `pending`：预留状态，当前原型中较少直接暴露
- `processing`：已接收，正在解析
- `success`：已进入可搜索状态
- `failed`：解析失败，显示失败原因

## 4. 搜索页 / Search

### 页面目标

让用户通过关键词和基础筛选快速找回目标内容。

### 主要模块

1. 搜索输入框
2. 来源类型筛选
3. 状态筛选
4. 标签筛选
5. 时间范围筛选
6. 搜索结果列表
7. 右侧检索面板

### 核心交互

#### 4.1 关键词搜索

- 用户输入关键词后执行搜索
- 在标题、正文、标签中查找匹配
- 结果按匹配程度与导入时间排序

#### 4.2 基础筛选

支持组合筛选：

- 来源类型
- 解析状态
- 标签
- 时间范围

#### 4.3 结果展示

每条结果至少展示：

- 标题
- 命中片段
- 标签
- 来源类型
- 时间信息
- 文件名或 URL

#### 4.4 空结果态

无结果时，展示：

- 未找到匹配内容
- 更换关键词建议
- 可点击的建议关键词

---

## 5. 详情页 / Detail

### 页面目标

帮助用户确认“这是不是我要找的那条内容”。

### 主要模块

1. 头部元信息
   - 状态
   - 来源类型
   - 导入时间

2. 主体内容区
   - 标题
   - 标签
   - 正文内容

3. 元信息侧栏
   - 来源方式
   - 文件名
   - 原始链接
   - 标签编辑
   - 时间线

4. 相关资料区

### 核心交互

#### 5.1 按标题回搜

- 用户点击按钮后，带着标题关键词返回搜索页

#### 5.2 标签编辑

- 用户可直接修改标签文本
- 保存后写回本地状态

#### 5.3 失败态重试

- `failed` 状态下展示重试按钮
- 点击后重新进入 `processing`

## 6. 数据对象语义

当前原型围绕统一 `document` 对象展开，关键字段包括：

- `id`
- `title`
- `content_raw`
- `content_clean`
- `content_preview`
- `source_type`
- `source_origin`
- `source_url`
- `file_name`
- `mime_type`
- `tags`
- `created_at`
- `imported_at`
- `parse_status`
- `parse_error_message`

这些字段已经足以为后续技术方案提供基础对象模型。

---

## 7. 原型特有行为

以下行为是原型演示逻辑，不应直接等同于正式系统实现：

- `localStorage` 持久化
- 内置种子数据
- 定时轮询式状态刷新
- PDF 成功 / 失败的模拟判定
- 搜索结果直接在前端内存中过滤

这些行为在技术方案阶段应被后端服务、任务队列、数据库和索引替代。
