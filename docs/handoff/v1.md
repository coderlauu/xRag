# v1 Handoff

**日期：** 2026-03-31  
**版本：** `v1 / Phase 1A`  
**状态：** archived

---

## 1. 当前版本一句话目标

验证 `采集 -> 存储 -> 检索 -> 找回` 的最小闭环，确认用户是否愿意持续导入资料，并能在后续通过关键词稳定找回。

---

## 2. 本轮边界

### 必须实现

- 导入页 / 搜索页 / 详情页三页闭环
- 手动文本保存
- 文件上传
- 解析状态展示
- 关键词搜索
- 基础筛选
- 详情查看
- 空结果态与失败态

### 明确不做

- AI 问答
- 语义搜索
- 真实 OCR
- 浏览器插件
- 团队协作
- 移动端

### 原型 / 文档中存在但本轮只做预留

- 链接正文抓取
- 扫描版 PDF OCR
- 标签系统的 ADR 收敛方向
- 排序增强

## 6. 输出要求

当 coder / implementer 基于本版本输出技术方案时，至少应：

- 先总结当前版本的真实需求边界
- 区分“原型演示行为”和“真实系统能力”
- 给出技术方案拆解与模块边界
- 给出技术选型建议及原因
- 明确数据模型、API、异步处理、搜索方案
- 区分 `v1` 必须实现与后续预留能力

---

## 7. 技术方案必须回答的问题

1. 文档对象如何持久化与索引化
2. 文件解析与状态流如何由后端任务驱动
3. 关键词搜索如何实现且便于后续升级到更强检索能力
4. 标签系统是否按 ADR-001 继续演进
5. 原型中的本地状态和 mock 数据如何替换为真实服务

---

## 8. 已知风险 / 历史决策

- 当前 `v1` 是静态 HTML 原型，不是正式工程实现
- PDF 成功 / 失败逻辑目前是模拟行为
- `ADR-001` 已记录标签系统演进方向，但尚未完全落到当前 `v1` 原型
- 真实 coding 前仍需补齐 `tech/architecture`、`tech/data-model`、`tech/api`
