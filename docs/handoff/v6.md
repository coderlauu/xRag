## 2. 本轮边界

### 必须实现

- 以 [Phase 2C PRD](/Users/coderlauu/xRag/docs/prd/2026-04-16-xrag-phase-2c-prd.md) 和 [Phase 2C Product Backlog](/Users/coderlauu/xRag/docs/prd/2026-04-16-xrag-phase-2c-backlog.md) 为准，冻结 `v6 / P0`
- 以 [Phase 2C Contract Freeze](/Users/coderlauu/xRag/tech/architecture/2026-04-16-phase-2c-contract-freeze.md) 为最新 contract source-of-truth
- 固定 `Lane 0` 的 `schema / shared-types / DTO / OpenAPI / SDK` 主线程实现顺序
- 切分后续 API read model、ingestion、web ops board 和 test lanes
- 更新 `current.md / status / exec plan`，保证后续新 session 可恢复

### 明确不做

- 在 `implementation-freeze` 完成前直接进入并行 implementation lanes
- 未经 `implementation-freeze` 或显式 contract 变更流程就修改 `schema / shared-types / API contract / citation / scope / eval` 主边界
- 把目标环境 `not_indexed` 文档回补默认改写为本轮产品主线
- 开放互联网联网回答
- 团队协作 / 权限模型
- 浏览器插件
- 移动端
- 全自动 agent 工作流

### 原型 / 文档中存在但本轮只做预留

- `P1-01` Drill-down 与样本回放
- `P1-02` 成本与质量联动分析
- 更深的知识库管理、摘要、推荐、多模型路由能力
- 任何需要重新打开 `Phase 2A / 2B` 已冻结信任边界的方向
