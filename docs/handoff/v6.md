## 1. 当前版本一句话目标

在 `Phase 2B` 已完成的可信问答与可控问答基线上，启动 `v6 / Phase 2C` 的范围规划与需求冻结，先明确“下一步真正要做什么”，再决定是否进入技术方案评估与实现。

---

## 2. 本轮边界

### 必须实现

- 明确 `v6 / Phase 2C` 的产品问题定义与验收标准
- 从 `Phase 2B` 已完成能力、剩余 backlog、运行态反馈中收敛下一版本候选范围
- 产出本轮 `PRD / backlog / interaction delta / technical tradeoff` 的第一版事实源
- 判断是否进入 `technical-evaluation` 与后续 `contract-freeze`
- 更新 `current.md / status / exec plan`，保证后续新 session 可恢复

### 明确不做

- 在 `PRD / backlog / interaction delta / technical tradeoff` 形成前直接进入实现 lane
- 未经新版本技术评估就修改 `schema / shared-types / API contract / citation / scope` 主边界
- 把目标环境 `not_indexed` 文档回补默认并入本轮产品范围
- 开放互联网联网回答
- 团队协作 / 权限模型
- 浏览器插件
- 移动端
- 全自动 agent 工作流

### 原型 / 文档中存在但本轮只做预留

- `Phase 2B` 的 `P1-01` 运维 / 评估面板增强
- 更深的知识库管理、摘要、推荐、多模型路由能力
- 任何需要重新打开 `Phase 2A / 2B` 已冻结信任边界的方向
